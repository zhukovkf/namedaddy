# Telegram-бот проверки названия бренда

Принимает название, ищет похожие товарные знаки в базе (открытые данные
Роспатента), показывает классы МКТУ и статус найденных знаков, и выдаёт
эвристическую оценку вероятности успешной регистрации.

## Как это устроено

- `db.py` — SQLite-хранилище + FTS5-индекс для быстрого отбора кандидатов.
- `ingest.py` — импорт реального CSV-дампа открытых данных Роспатента.
- `load_sample.py` — загрузка небольшого набора **вымышленных** демо-записей,
  чтобы сразу проверить бота без скачивания полного реестра.
- `similarity.py` — нормализация текста (в т.ч. похожие кириллица/латиница),
  упрощённый фонетический ключ, пересечение классов МКТУ, вес статуса знака,
  композитный риск-скор → вероятность регистрации.
- `bot.py` — сам бот на aiogram 3.

## Важная оговорка

Оценка вероятности — это эвристика на основе текстового/фонетического
сходства и пересечения классов МКТУ, **не юридическое заключение**. Реальное
решение Роспатента учитывает семантику, визуальное восприятие, известность
бренда и практику Палаты по патентным спорам. Для реальной подачи — нужен
патентный поверенный.

## Быстрый старт (на демо-данных)

```bash
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python load_sample.py          # грузит демо-данные
cp .env.example .env            # впишите токен от @BotFather
python bot.py
```

Проверьте в Telegram: отправьте боту, например, «Соколёнок» — он найдёт
похожие демо-записи «Сокол» и «СоколЪ Трейд».

## Подключение реальных данных Роспатента

1. Откройте страницу открытых данных:
   https://rospatent.gov.ru/opendata/7730176088-tz
2. Скачайте актуальный CSV (`data-YYYYMMDD-structure-YYYYMMDD.csv`) и файл
   структуры (`structure-YYYYMMDD.csv`) — **обязательно сверьте названия
   колонок в структуре**, они периодически меняются между релизами.
3. В `ingest.py` поправьте словарь `COLUMN_MAP` под реальные заголовки.
4. Запустите импорт (файл большой — читается чанками):
   ```bash
   python ingest.py path/to/data-YYYYMMDD.csv --delimiter ";" --encoding utf-8
   ```
5. Готово — `bot.py` начнёт искать по полной базе.

Полный реестр — это ~1-2 млн записей и несколько гигабайт, импорт может
занять от нескольких минут до часа в зависимости от машины.

### Дополнительно стоит учитывать

- Основной открытый реестр обычно содержит преимущественно действующие
  регистрации; сведения о досрочном прекращении публикуются Роспатентом
  отдельно — для точного статуса «прекращён» имеет смысл дополнительно
  подгрузить и этот реестр (см. страницу `/opendata`).
- Помимо реестра **регистраций**, у Роспатента есть открытые данные по
  **поданным заявкам** (ещё не зарегистрированным знакам) — они тоже являются
  препятствием для регистрации нового знака и их стоит подключить отдельным
  импортом с `status="pending"`.
- Для больших объёмов и постоянной нагрузки лучше перейти с SQLite на
  PostgreSQL + расширение `pg_trgm`/`rum` для триграммного поиска.

## Деплой

### Вариант A — VPS (long polling)

Простейший вариант — VPS + `systemd`/`tmux`/Docker с long polling
(`dp.start_polling`, файл `bot.py`).

### Вариант B — Render.com (бесплатно, без своего сервера)

Render не даёт бесплатный "always-on" процесс, зато даёт бесплатный Web
Service, который поднимается по входящему HTTP-запросу — для этого в
проекте есть `webhook_app.py` (та же логика бота, но через вебхук вместо
long polling) и `render.yaml`.

1. Залейте все файлы проекта в свой GitHub-репозиторий (GitHub → Add file →
   Upload files → перетащите файлы → Commit).
2. Зайдите на https://render.com → Sign in with GitHub (аккаунт уже
   авторизован — просто подтвердите доступ к репозиторию).
3. New → Blueprint → выберите репозиторий — Render подхватит `render.yaml`
   и настроит сервис автоматически.
   (Если не подхватит: New → Web Service → выберите репозиторий → Build
   Command: `pip install -r requirements.txt && python load_sample.py`,
   Start Command: `python webhook_app.py`.)
4. В настройках сервиса → Environment → добавьте переменную `BOT_TOKEN` со
   значением токена от @BotFather (впишите прямо в дашборде Render, не
   нужно передавать его куда-то ещё).
5. Deploy. После первого запуска бот сам зарегистрирует вебхук в Telegram
   (см. `on_startup` в `webhook_app.py`) — можно писать боту.

Free-план Render «засыпает» после ~15 минут без запросов и просыпается за
несколько секунд на первое сообщение — для личного/тестового бота это
нормально; для бота с постоянной нагрузкой лучше платный план или VPS.

## Ограничения текущей версии

- Фонетический алгоритм — упрощённая эвристика для русского языка, не
  формальный лингвистический стандарт.
- Не учитывается изобразительная часть знака (логотип), только словесная.
- Не учитывается репутация/известность бренда (для общеизвестных знаков
  Роспатент применяет расширенную охрану даже вне их классов МКТУ).
